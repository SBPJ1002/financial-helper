1. Visao Geral

O Financial Helper e uma plataforma completa de gestao financeira pessoal construida como monorepo full-stack. Integra-se com o Open Finance brasileiro via Pluggy para importar automaticamente transacoes bancarias, investimentos e saldos. Oferece um assistente de IA (Google Gemini), simulador de investimentos com juros compostos, acompanhamento de evolucao de gastos, e dashboard analitico com graficos.

O sistema trabalha com dados brutos do Open Finance de forma generica — nao depende de um banco especifico. Ate o momento, o teste foi realizado com o banco XP.

2. Arquitetura e Todos os Arquivos

Visao Geral da Arquitetura

Monorepo com dois projetos independentes (client/ e server/) que se comunicam via API REST. O frontend usa Vite como bundler com proxy para /api apontando para localhost:3001. O backend usa Express 5 com service layer pattern. PostgreSQL rodando via Docker Compose.

Client: porta 5173 (dev) Server: porta 3001 PostgreSQL: porta 5432 pgAdmin: porta 5050

6. Paginas do Frontend — Detalhamento

As 15 paginas estao listadas na secao 2 com seus arquivos. Aqui o detalhamento das funcionalidades de cada uma:

Dashboard.tsx

Consome GET /api/dashboard/summary?month=YYYY-MM. Exibe 5 stat cards (receita total, despesas fixas, despesas variaveis, saldo, total investimentos). Grafico de barras Recharts com getPreviousMonths(6) mostrando receita vs despesa. Pizza breakdown por categoria com nome e emoji. Top 5 gastos do mes. Se ha contas bancarias, mostra resumo com nome/tipo/saldo.

Expenses.tsx

Pagina mais complexa do frontend (~800 linhas). Estado month com navegacao (< Fevereiro 2026 >). 3 abas com contagem: Income (N), Fixed (N), Variable (N). Cada aba tem tabela com colunas: Descricao, Valor (formatCurrency), Data (formatDate), Vencimento (dueDay), Categoria (nome+emoji), Status (badge PAID/PENDING), Pagamento (badge CREDIT/DEBIT/PIX), Acoes (editar/deletar). Badge de parcela: se totalInstallments > 1, exibe "(1/2)" ao lado da descricao. Filtros: busca por texto, categoria, ordenacao (valor asc/desc, data, nome). Modal CRUD com form dinamico por tipo. Export CSV. ConfirmDialog para deletar.

Investments.tsx

Form dinamico baseado no InvestmentType selecionado. Renda Fixa: campos de rendimento (yieldDescription), data de vencimento, taxa. Tesouro: titulo, taxa, indice (SELIC/IPCA). Renda Variavel: busca de ativo (ticker), quantidade, preco medio. Graficos: pizza por categoria (Fixed Income/Variable Income/Other), barra horizontal por tipo de investimento. P&L com badges verdes (lucro) ou vermelhos (perda). Botao "Atualizar Cotacao" individual ou "Atualizar Todas" em lote.

Banking.tsx

Verifica isPluggyEnabled via GET /api/banking/available. Se habilitado: widget Pluggy Connect para conectar banco, lista de conexoes com status (ACTIVE/ERROR), botao sync por conexao, tabela de transacoes com paginacao (page/limit), botao "Importar Transacoes" que chama POST /api/banking/import e exibe resultado (N receitas, N despesas, N ignoradas, erros). Se nao habilitado: instrucoes de setup.

Evolution.tsx

Lista despesas fixas que tem historico mensal. Para cada: grafico de tendencia inline (sparkline), indicador de tendencia (↑ crescente / → estavel / ↓ decrescente) calculado pela funcao getTrend(). Modo comparacao: seleciona ate 4 despesas, compara em line chart Recharts. Metas: botao "Definir Meta" que cria ExpenseGoal. Projecao: media movel ponderada de 6 periodos para estimar valor futuro.

15. Inteligencia Artificial

Adapter Pattern Multi-Provider

O ai.service.ts implementa um adapter pattern que permite trocar entre provedores de IA. Atualmente Google Gemini esta implementado e configurado. O SDK Anthropic esta instalado como dependencia para uso futuro.

Auto-deteccao de Modelo Gemini

detectGeminiModel(apiKey) busca GET /v1beta/models?key={apiKey}, filtra modelos que suportam generateContent, exclui especializados (TTS, imagem, robotics, nano, gemma, embedding, deep-research), e retorna o primeiro match da lista de preferencia:

1. gemini-3-flash2. gemini-3-pro3. gemini-2.5-flash4. gemini-2.5-pro5. gemini-2.0-flash6. gemini-1.5-flash7. gemini-1.5-pro

Cache de 5 minutos. Google Gemini usa ?key= no query param (NAO Bearer token).

System Prompt Dinamico

O promptBuilder.service monta prompt em PT-BR com: receitas, despesas fixas + historico, despesas variaveis do mes, totais, investimentos com P&L, taxas BCB, contas bancarias, 30 ultimas transacoes, e analise de tendencia (gastos crescentes, items acima do limite).

Tratamento de Erros

  • 400: Retry sem systemInstruction (modelos antigos)
  • 429: Rate limit — mensagem amigavel
  • 403: Acesso negado — verificar billing
  • Safety block: Mensagem sobre bloqueio de seguranca
